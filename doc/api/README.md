## Concepts Reference

### Core Concepts

- `Aggregate` - Types that are aggregates (arrays or classes with no special members)
- `TriviallyCopyable` - Types safe for bitwise copy
- `StandardLayout` - Types with C-compatible memory layout
- `TrivialType` - Types that are both trivially copyable and trivially default constructible

### Serialization Concepts

- `SafelySerializable` - Types safe for binary serialization
- `SerializationCompatible<T, U>` - Type pairs with compatible layouts
- `PlatformIndependent` - Types with platform-independent layout

## Module Groups

The API is organized into the following groups:

- **core_api** - Layout signature generation and comparison
- **core_concepts** - Type classification concepts
- **serialization_api** - Serialization safety checking
- **utility_concepts** - Utility type concepts
