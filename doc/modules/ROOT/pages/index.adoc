= Boost.TypeLayout
:navtitle: Home

Compile-Time Memory Layout Analysis using {cpp26} Static Reflection (P2996)

[.lead]
Boost.TypeLayout is a header-only {cpp26} library that provides compile-time memory layout analysis and verification. It generates human-readable signatures that uniquely identify type memory layouts, enabling robust binary interface verification and ABI compatibility checking.

[NOTE]
====
*Core Guarantee*: Identical signature ‚ü∫ Identical memory layout
====

== Quick Example

[source,cpp]
----
#include <boost/typelayout.hpp>
using namespace boost::typelayout;

struct Point { int32_t x, y; };

// Automatic signature generation
constexpr auto sig = get_layout_signature<Point>();
// Result: "[64-le]struct[s:8,a:4]{@0[x]:i32[s:4,a:4],@4[y]:i32[s:4,a:4]}"

// Compile-time layout verification
TYPELAYOUT_BIND(Point, "[64-le]struct[s:8,a:4]{@0[x]:i32[s:4,a:4],@4[y]:i32[s:4,a:4]}");

// Serializability checking
static_assert(Serializable<Point>);

// Template constraints using concepts
template<typename T>
    requires LayoutMatch<T, "[64-le]struct[s:8,a:4]{@0[x]:i32[s:4,a:4],@4[y]:i32[s:4,a:4]}">
void process_point(const T& p) { /* ... */ }
----

== Key Features

[cols="1,2"]
|===
| Feature | Description

| *Layout Signatures*
| Automatic compile-time generation of portable layout descriptions

| *Platform Detection*
| Architecture and endianness encoded in signatures

| *Serialization Safety*
| Identify non-serializable types (platform-dependent, pointers, bit-fields) at compile time

| *Dual-Hash Verification*
| FNV-1a + DJB2 for ~2^128 collision resistance

| *{cpp} Concepts*
| `Serializable`, `LayoutCompatible`, `LayoutMatch` constraints

| *Zero Runtime Cost*
| All analysis happens at compile time
|===

== Requirements

* *Compiler*: Bloomberg Clang P2996 fork (or any compiler supporting P2996)
* *Standard*: {cpp26} with `-std=c++26 -freflection -freflection-latest`
* *Platform*: 32-bit or 64-bit, little-endian or big-endian
* *Floating-point*: IEEE 754 required
