## 1. Analysis Phase

- [x] 1.1 统计当前签名中字段对齐信息的冗余度
  - 分析 `test/` 目录中的签名示例
  - 计算冗余字节数和占比
  - **结论**：每个字段约 9 字符冗余 (如 `[s:4,a:4]`)

- [x] 1.2 识别必须保留字段对齐信息的场景
  - `alignas` 指定的特殊对齐
  - 复合类型（struct/class/union）字段
  - 位域（bitfield）
  - 数组类型
  - **结论**：以上场景需要，但偏移量已隐含对齐结果

- [x] 1.3 识别可以省略字段对齐信息的场景
  - 固定大小基本类型（i8, i16, i32, i64, u8, u16, u32, u64）
  - 固定大小浮点类型（f32, f64）
  - 固定大小字符类型（char, char8, char16, char32）
  - 布尔类型（bool）
  - 平台相关但可推导类型（ptr, ref, fnptr）
  - **结论**：技术上可省略，但保留有自文档价值
