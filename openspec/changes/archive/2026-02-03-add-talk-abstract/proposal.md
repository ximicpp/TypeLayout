# Change: 添加技术演讲摘要

## Why

项目缺少一份精炼的、面向技术会议审稿人的演讲摘要（Abstract）。C++ 技术会议的审稿人是经验丰富的 C++ 专家，他们评估提案时关注：

1. **技术创新性** - 是否展示了新技术的实际应用
2. **实用价值** - 是否解决了真实的工程痛点
3. **演讲吸引力** - 是否能引发听众兴趣和讨论

TypeLayout 是 P2996 静态反射的"杀手级应用"之一，需要一份能够吸引审稿人注意力的摘要。

## What Changes

- 新增 `doc/talk/abstract.md` - 技术演讲摘要（300-500 字）
- 新增 `doc/talk/outline.md` - 演讲大纲（可选，用于审稿人了解完整结构）

## Impact

- Affected specs: `documentation`
- Affected code: 无代码变更，仅文档
- 无破坏性变更

## 摘要策略

### 技术会议审稿人关注点

| 维度 | TypeLayout 的回应 |
|------|-------------------|
| **新颖性** | P2996 首个实用库级应用，展示静态反射的威力 |
| **痛点解决** | ABI 地狱、版本不匹配导致的静默损坏 |
| **Zero-cost** | 完全编译时，零运行时开销 |
| **无 IDL** | 无需 protobuf/flatbuffers 的代码生成 |
| **C++26 前沿** | 展示即将到来的语言特性的实际价值 |

### 核心信息

**一句话定位**: 
> "TypeLayout: Compile-Time ABI Verification via P2996 Static Reflection"

**核心价值主张**:
> 使用 C++26 静态反射，在编译时自动生成类型内存布局签名，
> 为共享内存、网络协议、插件系统提供零成本的 ABI 兼容性保障。

### 演讲类型建议

- **Regular Talk (60 min)** - 深入技术细节 + 现场演示
- **Short Talk (30 min)** - 概念介绍 + 核心用例
