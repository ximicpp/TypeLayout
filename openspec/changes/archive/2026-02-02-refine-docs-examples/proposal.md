# Change: 精简文档和示例，聚焦核心功能

## Why

经过 Nano 架构精简后，文档和示例需要与代码同步更新：
1. README 中的项目结构与实际不符（列出了已删除的文件）
2. 示例文件可以进一步精简，聚焦核心场景
3. doc/ 目录内容过多，许多页面已过时

## What Changes

### 1. 更新 README.md
- 修正项目结构描述
- 精简 API 文档，移除已删除功能
- 聚焦 3 个核心使用场景

### 2. 精简 example/ 目录
- 保留核心示例，确保每个示例都能编译运行
- 删除冗余或重复的示例

### 3. 精简 doc/ 目录
- 删除过时或冗余的文档页面
- 保留核心用户指南和参考文档

## Impact

- 影响文件: README.md, example/*, doc/*
- 不影响核心代码
